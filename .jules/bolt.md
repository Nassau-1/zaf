## 2025-02-19 - Removed Synchronous Subprocess from Node API Endpoint
**Learning:** In `dashboard/server.js`, the `/api/data` endpoint synchronously executed `runParse()` using `execSync` for every HTTP GET request. This blocked the entire Node.js event loop, preventing other requests from being handled until the subprocess finished parsing markdown files. Since `chokidar` is already watching the filesystem and running `runParse()` asynchronously when files change, the synchronous parse on read was entirely redundant and caused severe performance degradation on load.
**Action:** Removed `runParse()` from the `/api/data` request handler. For future codebases, always verify that API endpoints do not execute blocking, synchronous subprocesses (`execSync`), especially when reactive file watchers are already maintaining state.
